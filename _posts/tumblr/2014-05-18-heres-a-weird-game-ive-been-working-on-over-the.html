Here’s a weird game I’ve been working on over the weekend.

It started as a recreation of the Glove Puncher game I posted earlier, but it kind of evolved into a shoot-em-up styled game. It’s very hard to explain. I hope I can actually see this idea to completion, I really want to start pushing games to the Google Play store (with or without advertisements).

In terms of programming, I really focused on state-based object logic. Rather than determining action based on several to many different properties, each object can be only in one state, which is an enumerated value. For example, the blue rocks in the pictures have states like FALLING, COLLECTED, EXITED, and so on. Each object’s states are publicly available, and the current state it’s in can be retrieved. The main game’s update loop handles object actions based solely on state - it’s much more predictable and manageable. In terms of encapsulation I’m not sure how formal it is to have something like states publicly available…but it’s a prototype, anyway.
