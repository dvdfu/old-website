My current inactivity is largely due to final exams, which are over in a few days. But between exams I’m still trying to work with libgdx.

Today I worked with the browser port for libgdx games. I still don’t fully know how to export browser-ready games, and I probably won’t be able to find out until after I have my own server to upload files onto. I can still compile and test games in web browsers, but it is somewhat problematic:

  • libgdx uses WebGL to run its games, which is not supported by the latest versions of Firefox
  • On Chrome, the plugin must be installed, and even then I had to disable settings to allow WebGL to run
  • When it’s working, the program is incredibly unresponsive and updates frames only every few seconds
  • The screen isn’t fully rendered in this case (the two missing tiles in the top-right corner are present on the desktop version)

Doing some research, the HTML game must be fully compiled/exported/hosted on a server, otherwise its performance while testing is incredibly sluggish. However, even exporting such a simple game takes between 2-3 minutes, so it’s not feasible to export whenever I want to test. Luckily, I can still use the desktop version to test.

The program I wrote is a very simple. It creates an array (one-dimensional) of colors, draws square tiles of a given width according to the colors in the array, and updates each tile’s RGB component based on its proximity to the input coordinates (in this case, the mouse).

I want to distribute any simple or experimental games I make onto web browsers: I think they are the most accessible and will have the largest audience. I will have to look more into libgdx and WebGL, first.
