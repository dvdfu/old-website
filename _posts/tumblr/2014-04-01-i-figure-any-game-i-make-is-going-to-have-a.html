I figure any game I make is going to have a pixelized 2D appearance, so I went out to look for a good sprite editor. Lightweight-ness and functionality were priorities. I found mtPaint: it did the job, so I’m sticking with it.

These two sprites are recreations of sprites from Bloberynth, the first game I ever made (on Game Maker, when I was 11). I feel like these iconic tiles will become stock images for testing in the future.

My ventures into libgdx were minimal today, but I did look into packing textures. TexturePacker2 is a built in libgdx tool that stores every image in a given directory into a “packed” image that requires less memory to load. This greatly optimizes speeds in games that have a large library of assets.

It’s a very neat tool. I need to learn how to import the packed image and create libgdx textures out of it. It’s my next step for making shiny-looking games.
