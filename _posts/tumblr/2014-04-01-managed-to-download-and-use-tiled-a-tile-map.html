Managed to download and use Tiled, a tile map editor. Tiled exports files in a .tmx format, which is supported by libgdx. I most likely won’t need it until long into game development, but I might as well start now.

I also got gdx-texturepacker.jar, a UI for the texture packing program, understand using animation classes, and have set the icon for the android app.

I’m now running into problems with file handling: since each version (desktop, android) of the game exists in a separate folder, they have different directories relative to the location of game assets, so I have some problems detecting files. This wasn’t a problem in Drop because the project was not created using the libgdx setup UI. I believe the answer lies somewhere in folder redirection.

Researching helps a lot, but there are some things that just can’t be learned without repetition. I think I’m going to run into that a lot while making the games.
