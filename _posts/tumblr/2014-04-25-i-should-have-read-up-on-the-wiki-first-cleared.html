I should have read up on the wiki first - cleared up a lot of information, especially about image rendering terminology. For future reference, I’m rephrasing here:

  • A Texture is a decoded image file with image size dimensions in powers of two. It is uploaded to the GPU
  • A TextureRegion is cropped portion of a texture; both TextureRegions and Textures can be drawn. Texture portions can be manually drawn using SpriteBatch functions, but TextureRegions save a little work
  • Sprite further extends from TextureRegions in that they can be rotated, tinted, blended, etc.
  • All of the above (and others) are drawn using SpriteBatch, which is essentially a single image + reference information that can more efficiently draw images than having each separate image uploaded and called from the GPU
  • TextureAtlas is used to load images from the file system that have already been packed into a large “atlas” image, the benefit being the initial resource loading is more optimized.

Anyway, I’ve spent most of the past few days learning more than anything. It turns out camera operations are fairly straightforward for panning and zooming - I just have to declare variables for camera coordinates and zoom, so I can render GUI related things properly. I had my head wrapped around the idea that camera functions required a lot of matrix calculation, but that is all hidden through its implementation.

I also looked into Bitmap Fonts. Heiro is a built-in libgdx tool that converts system fonts into Bitmap Fonts. Bitmap Fonts are packaged very similarly to Texture Atlases, but they also contain information in a .fnt file that associates each character image to an ASCII character. Bitmap Fonts are very lightweight and rasterized, making them perfect for 2D video games (but little else: they can’t be variably stretched, have effects added, and so on).

While it is very easy to use Heiro and convert pre-existing fonts into usable Bitmap Fonts, I’m having a very hard time finding a program that will let me design my own Bitmap Fonts from scratch. As you can see in the second image, I have an alphabet just waiting to be used! But the programs I’ve come across are either not free, not supported by Linux, or incredibly outdated. It is possible, but VERY time consuming to write the .fnt file manually. Maybe I’ll write my own program! But it’s not a priority right now.
