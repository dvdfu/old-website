I haven’t touched upon the particle effect generator, but there is a convenient interface named Poolable. The game knows to pull Poolable objects out of a ‘pool’ before resorting to making new ones. This way, objects can recycled, reducing memory use. This works perfectly for objects that are created in large amounts or with high frequency; i.e. particles, bullets, effects…

This is exactly what I used when I made the flashy sparkles in the images above (I really love using particles). The game is a grid-based puzzle game that is themed around gems. The goal of the game is to collect every gem in a given level - no extras. There are many ways to ‘collect’ gems, and many other gameplay elements that I am working on and won’t really go into at the moment.

Also, I used really simple placeholder sprites. I plan to make the real game look really pretty - possible with painted images instead of low resolution sprites like I usually use.

I hope this will be the first project I see to completion. I picked something a little less complicated to program (although it turns out, it’s still somewhat complicated). It’s being developed for Android and desktop, but mainly Android.
