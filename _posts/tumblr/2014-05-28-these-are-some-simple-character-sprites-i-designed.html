These are some simple character sprites I designed for no reason other than to see what I’d come up with. I won’t lie, Cave Story inspired me, but there are only so many ways you can draw a cartoonish humanoid in a 16x16 pixel sprite.

Game development has slowed down a bit because I’ve been taking some time to learn front-end web development. It’s a useful skill to have and great way to present yourself online.

I’m also reconsidering my entire approach to libgdx: I hadn’t realized the importance of Scene2D package. It abstracts away object representation on the screen, making object handling and UI much simpler.

For a while, a big problem for me was figuring out how to handle object detection. Up until now I had array lists of objects which the game would loop through every tick to check for collisions. The platformer mock-up I posted earlier had very refined collision, but I knew it would break if I used the same system for, say, a top-down shooter, which would have hundreds of bullets flying everywhere. I had to start looking into quad-trees and ways to make collisions more efficient, while also holding all objects in a data structure that could somehow sort objects based on depth so that they would render in the proper order. I know of ways to do this but their implementations would be nightmarish for me and I wasn’t confident in their run time.

So for now I’m going to learn how to use Scene2D. I will return to more low-level implementations when I have a stronger grasp of those concepts.
