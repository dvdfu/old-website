Cool stuff today!

After a bit more reading, maybe Box2D isn’t the best idea for a 2D platformer. I got it to do some really cool physics stuff, but overall it’s a lot more than required for simpler collisions. So I got rid of all that and rewrote really simple game object classes (along with classes for animation and input). I borrowed the neat little “debug rendering” from Box2D, which you can see in the picture above.

Plus, I made a somewhat more optimized map loader: instead of placing each block as its own square tile, the loader scans for successive rows and columns of blocks, and adds a bigger block in as one object. The extra loading time is a non-factor and I can picture this significantly reducing collision operation times or even glitchy collisions.

Then I finally got Github to do what I want, so overall a really good day for progress.

On one last note, here is a working rendering order, since up until now rendering has been flickering or delaying:

  • register key inputs
  • move and THEN update character
  • move and THEN update camera
  • set tiled map renderer to camera
  • update sprite/shape batches using camera matrix
  • clear screen
  • draw objects, front-most objects last
