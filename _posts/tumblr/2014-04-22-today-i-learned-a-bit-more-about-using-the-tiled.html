Today I learned a bit more about using the Tiled and Box2D tools in libgdx. A did a quick exercise to load maps:

  1. Using Tiled, I created very simple tiled maps (on the right) andÂ saved the .tmx and tileset image files into the game assets folder
  2. I read the files and iterated through each tile on a specified tile layer. Non-null tiles were given “bodies” and added to the “world”
  3. The map is drawn using a camera and the Box2D debugger renderer, i.e. there were no sprites used, only the outlines as specified when I define the body shapes

There is a much more in-depth implementation of Box2D which involves specifying metrics, body properties, and more things I would use if I were interested in a physics-heavy game.

I neglected using a tool that chains connected bodies, which would be useful for solid platformer blocks in avoiding “ghost” collisions. It’s something I’ll approach when I have a greater understanding of Box2D terminology. Box2D seems like a pretty useful thing to learn, especially considering it’s cross-platform, so I’ll have that if I ever jump to Flash or C++. Apparently games on the Wii and DS have used Box2D, if you can trust Wikipedia.
