Screenshots that show off really simple OpenGL shaders. One of the reasons I chose to use libGDX is to have fine control over my game’s programming. Shaders are a really big part of that so I finally put some time into learning how to use them.

libGDX lets you write C-style shaders in OpenGL’s shader language and are used in Java when read as a file or, strangely, entered as a Java string. Every pixel is processed through the shader pipeline by the GPU before being rendered. I render pixels in the first picture with lower opacity, so every new frame blends with the last, creating a blur effect, and with averaged R/G values in the second picture, casting a yellow hue. These are very dull examples, but with enough practice you can do some prettyÂ amazing things. Learning how to write good shaders is not my priority right now, but definitely a long-term goal.

Slightly related, I learned how to use frame buffers. Previously, the way I achieved a “2x zoom” effect was simply by stretching the sprite batch and sprites individually. While sprites are stretched perfectly, their positions are stored as floats, so spacing between individual sprites may be 1 pixel and break the illusion of a pure 2x zoom. Now everything is rendered at 1x to a frame buffer, and the buffer is scaled using a nearest-neighbour filter. Performance also noticeably increased this way.

In short, learning lower level processes in libGDX is both fun and useful
