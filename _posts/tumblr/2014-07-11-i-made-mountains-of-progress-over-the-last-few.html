I made mountains of progress over the last few days!Â I managed to devise a way to read and write level data. Level data looks like this:

name;8;10;03b;04b;05b;06b;07b;09sh;10bam;11…

It’s human readable, and possible to build levels straight from notepad (which is advised against). The real progress was that I made a built-in level editor. The tab key switches between edit/play (you can’t tell in the images). If I ever port this feature to mobile, I would add touch-based buttons to select tools, so it would look something like a paint program. The level editor even comes with undo/redo functions that I built with stacks.

In the images, you’ll see that I’m adding and changing the properties of blocks, and then immediately going live with the updates and playing the level. By “going live”, the game quickly saves the level data to a local directory and immediately loads/renders it.

On top of all of that, I looked into porting to HTML and hosting it on GitHub using gh-pages. From GitHub’s point of view this was a terrible idea, since the generated HTML files were in the tens of thousands of lines, and it rendered my repository statistics useless. The game seems to host fairly well, although it doesn’t run on Safari and has serious audio lag.
