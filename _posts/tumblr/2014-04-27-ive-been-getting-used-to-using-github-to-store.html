I’ve been getting used to using Github to store and manage my game projects. Unfortunately, that also means I don’t have a clue about what I’m doing. I accidentally erased all of Glove Puncher (though you can still download and play it). It’s very unfortunate. But it was also just a prototype of something much bigger; more on that later.

In terms of working progress, I fully learned animation and input processing implementation thanks to this guy. I’m a bit surprised these functions aren’t built in, but they’re quick fixes and allow for a ton of flexibility. Likewise, I made some moderate changes from the original source code provided. The screenshot is just a Box2D rectangle bouncing up and down, while cycling through 4 32x32 sprites. Also, the console outputs text on key presses (not holding keys, though) to test the input processor.

Lastly, but most importantly, I’m starting to understand what game development really entails. I’ve been reading up on some insightful posts around the Internet - Derek Yu, Tommy Refenes, /r/gamedev. Here’s a brief summary:

  • I used to look down on high level game engines such as Game Maker, etc. But, it’s painfully obvious that the end product is all that matters. Rolling out content is the most important, so whatever works, works.
  • Likewise, being able to show a fully finished product is your goal - how you got there isn’t what people will look for.
  • The definitive aspect of people who make games is the ability to have a vision and carry it through to the end. The quality of the game depends on everything else, but you can’t make games if you can’t meet your goals.
  • Hard work is so, so important. And it might not always be where you think it should go, either. A lot of it happens in planning - and after, polishing. An author may spend a long time writing a book, but most of the work is certainly not spent typing it. Same with video games.
  • Productivity helps with working hard. Github’s Contribution Activity is actually an incredible way of documenting productivity, and motivating, too.

A lot of that should come as intuitive, but it really changes what I think about game development - for the better.
